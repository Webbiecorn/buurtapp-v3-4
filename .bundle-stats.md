# Performance Monitoring Data

This file contains bundle size metrics tracked over time.
Generated by: `npm run build:analyze`

## Purpose

Tracks bundle size trends to detect:
- Unexpected size increases
- Large chunk additions
- Opportunities for optimization

## Format

```json
{
  "timestamp": "ISO date",
  "totalSize": 12345,
  "totalSizeFormatted": "123 KB",
  "assetCount": 47,
  "largestAssets": [...],
  "largeChunks": [...],
  "byType": {
    "javascript": 67890,
    "stylesheet": 12345,
    "other": 123
  }
}
```

## Usage

Run `npm run build:analyze` after builds to update metrics.
Compare with previous builds to track size changes.

**Note:** This file is git-ignored to prevent conflicts.
